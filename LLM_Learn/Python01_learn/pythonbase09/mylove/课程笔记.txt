			原码反码补码
			数字分为正数和负数
			存储数字的最高位为符号位 正为0 负为1
			正数的原码 反码 补码一致
			负数的原码 是符号位为1 其余位求2进制 就是原码
			负数的反码 是求出来的原码 符号位不变 其余位取反（1变0 0 变1）
			负数的补码 是求出来的反码 + 1 符号位不变
				
			
			看详细文档上的例子
			
			计算的时候一定要转成补码去计算  得到的结果也是补码，最终我们要的数字是原码
			注意：计算的时候符号位是参与计算的，但是我们自己求原码反码补码的时候符号位不变

	if分支
		分支的作用就是让代码可以选择性去执行不同的操作，执行完成后接着走后面的语句
		
		单分支
			if 值:
				if分支的代码块
			
			执行流程：
				先把值，默默滴心里面转化为布尔值，如果这个布尔值是真，那么就走对应的if分支代码块，如果为假，则不走if分支代码块。继续往下执行
		
		
		案例：
			给定天气，天气好就出去玩
			随机获取一个数，偶数就爱赵丽颖
			
		
		双分支
			if 值：
				if分支的代码块
			else:
				else分支的代码块
				
				
				
			执行流程：
				先把值转化为布尔值，如果为真，那么走if的代码块，else的代码块就不走了，直接跳楼，如果为假，那么if的代码块就不走了，直接走
				else的代码块。执行完成，继续往下走
			
		
		
		
		
		案例：
			给定天气，天气好就出去玩，不好在家写作业
			随机获取一个1到100的整数，偶数就爱赵丽颖，奇数就爱杨幂
		
		多分支
		
			if 值：
				if的分支代码块
			elif 值：
				elif的分支代码块
			....
			else:
				else的分支代码块
				
				
				
			执行流程：	
				先把if后面的值转化为布尔值，如果为真就去执行if的代码块，如果为假，走下面elif，把elif的值转化为布尔值，如果为真，
				走elif的分支代码块，如果为假，再走下一个elif,继续把值转化为布尔值，如果为真，走elif的代码块，如果为假继续走下一个
				elif,直到所有的elif值都是假，最后走else的代码块。 不管走了哪个分支，其余分支都不走了，直接跳楼走下面的代码
				
				
				注意：多分支else可以不写，也可以写，关键看你的业务逻辑
				
		
		案例：
			分数评价
			体重评分

				位置参数和关键字参数同时存在 那么1、位置参数在前，关键字参数在后  2、位置参数和关键字参数不能重复给一个形参赋值
				
				
			3、默认值参数，形参默认值

		函数传参传递可变数据和不可变数据的区别*****************
		
			函数实参在传递给形参的时候都是把值传递过去的，和给变量赋值道理一样，本质都是把一个数据在堆里面的地址给变量存储
			无论是可变类型还是不可变类型
			
			
			不可变数据   int  float  bool  str  None  tuple
				形参变量 = 新的数据   那么形参变量内部地址发生改变  不会影响外部实参数据变量地址
				形参变量 = 老的数据（数据相同）  那么形参变量里面的地址和实参变量里面的地址一样（优化）
				不可变数据也没办法修改容器内部的数据
				
			可变数据   list  dict  set
				形参变量 = 新的数据   那么形参变量内部地址发生改变  不会影响外部实参数据变量地址
				形参变量 = 老的数据（数据相同） 那么形参变量里面的地址和实参变量里面的地址不一样
				可变数据可以操作容器内部的数据，形参拿到实参数据的地址并不变，而是修改容器里面的数据，此时内外数据都会修改，因为地址没变
				
			
			特别注意1：元组元素如果存在可变类型，我们就完全可以把元组当做可变数据对待。一般不存在
			特别注意2：可变数据 *= 2  和 可变数据 = 可变数据 * 2  不同  一个是操作原容器  一个是创建新容器

	封装
		1. **隐藏内部实现**：把类的私有数据（属性）和操作数据的方法绑定在一起，外部无需知道类的内部如何存储、如何处理数据；
		2. **保证数据安全**：限制外部对属性的随意修改，通过统一接口做数据校验、日志记录、联动更新等，避免非法数据进入；
		3. **降低耦合度**：外部仅通过类暴露的接口和类交互，后续类的内部实现修改时，只要接口不变，外部代码无需任何调整。
		
		封装的本质是私有化属性数据（敏感）
		然后外面不让直接访问，保证私有数据的安全 无论是属性还是方法。
		外面如果要访问就必须提供对外的接口（方法）去访问
		
		
		要做封装就两步：1、属性私有化  2、提供接口
		
		
		
		
		
		
		封装的两种种方式（本质其实两种方式）
		
		
			1、约定私有 + 接口     单下划线私有属性  + 方法接口
			2、改名私有 + 接口     双下划线私有属性  + 方法接口
			
			3、接口式封装@property 本质其实就是简化方法接口 不能算作是一种封装
			
			
		python当中其实没有绝对的私有封装，不管怎么封装都能强行在外面访问。
			
			约定私有属性是单个下划线，在外面可以直接访问
			改名私有属性其实就是双下划线，在外面可以通过修改的名字直接访问
				类属性被修改为 类._类名__属性名
				实例属性被修改为 self._类名__属性名
				
		
		最佳实践：
			最小化封装，能不封装就别封装，如果非要封装，我们选择第二种封装 + 接口简化
			

		
		
	继承
		让一个子类（派生类）复用另一个父类（基类 / 超类）的属性和方法，
		同时子类可以根据需求扩展、重写父类的内容，实现代码复用和功能扩展，避免重复编写相同代码。（青出于蓝胜于蓝）
		
			
		继承的优点：
			1. 代码复用：父类的通用逻辑只写一次，所有子类直接复用，减少冗余；
			2. 功能扩展：子类可通过新增方法 / 重写方法快速扩展功能，符合 “开闭原则”（对扩展开放，对修改关闭）；
			3. 结构清晰：通过继承梳理类之间的层级关系，代码结构更直观。

		继承的缺点：

			1. 耦合度高：子类和父类强关联，父类的修改会直接影响所有子类，增加维护成本；
			2. 多继承复杂：多继承会导致 MRO、菱形继承等问题，代码可读性和可维护性下降；
			3. 灵活性低：继承是静态的，编译时确定，无法动态改变继承关系。
			
			
		
		单继承
			属性继承
				实例属性(不是无条件继承的)
					自己没有init 调用了父类的init，会继承
						子类调用了父类的init被称作实例属性继承，本质其实是调用父类方法，把父类的实例属性添加给子类实例
							子类实例添加了父类init当中的属性
							注意：如果父类里面出现__xxx私有，那么在没有调用前就改名为self._Parent__xxx,所以子类实例添加了一个_Parent__xxx
							
						最佳实践：如果子类没写init，继承父类，后期访问私有属性要不就使用接口，要不就使用修改的名字（不建议）
						
					
					自己有init，调用的是自己的init，不会继承
						子类有自己的init，代表子类不继承父类的实例属性，调用自己的init去添加子类实例属性
							子类实例添加了子类init当中的属性
							注意：如果子类里面出现__xxx私有，那么在没有调用前就改名为self._Child__xxx,所以子类实例添加了一个_Child__xxx
					
						不继承父类的实例属性，如果有相同属性，可以通过调用父类的init去做初始化，效果比自己重新写一遍同名属性更好
							super().__init__(相同的属性参数)
								注意点：父类如果有改名式私有可以认为有继承，因为通过改名的名字可以访问。但是不建议
								
							不同的部分自己写self.xxx=xxx
					
					
					
				类属性继承（无条件继承）
					类属性会继承父类的所有，如果子类没有同名属性，可以直接访问父类的
					如果子类有父类的同名属性，那么就访问子类的自己的
					子类和父类双下滑线私有类属性，都可以通过改名的方式让子类获取到，但是不建议

			方法继承
				实例方法（无条件继承）
					如果子类没有，那么就找父类，调用父类的实例方法，不过self传递的就是子类实例了
					如果子类有，那么就调用子类的方法，我们称为叫方法重写（子类当中有父类同名方法）
					注意：方法前面双下滑线开头叫私有方法，也是改名后的效果，此时不是方法重写
					
					
				类方法（无条件继承）
				
					如果子类没有，找父类直接调用父类的类方法，如果子类有就调用子类的同名类方法（类方法重写）
				
					注意：
						通过super().类方法()调用父类方法，那么cls传递的是子类
							super代表的是继承的父类（），调用类方法的时候传递的是当前正在运行的子类
							
						通过父类名.类方法()调用父类方法，cls传递的就是父类
				
				
				
				静态方法（无条件继承）
					如果子类没有，找父类直接调用父类的静态方法，如果子类有就调用子类的同名静态方法（静态方法重写）
					
					注意：
						静态方法不能通过在子类方法当中通过super().static_method直接调用，因为它默认要传递两个参数，而静态方法当中没有参数
						super() ===> super(本类，cls/self)，而静态方法当中没有cls/self
					
				
				
			
			总结：
				除了实例属性以外，剩余所有的成员都是无条件继承，也就是子类有就拿子类的去用，没有就找父类去拿

	生成器
		1、概念
			生成器（generator）是一个用于创建迭代器的简单而强大的工具。它的写法类似于标准的函数，但当它要返回数据时会使用 yield 语句，
			而不是普通函数的return语句。当在生成器函数中使用 yield 语句时，函数的执行将会暂停，并将 yield 后的表达式作为本次迭代的值返回。
			遇到return 生成器执行会强制结束，即使后面还有yield要生成的数据也会立即结束，而return的结果会作为异常StopIteration的value值
			

			生成器（Generator）可以看成是Python中一种特殊的迭代器，它也有`__next__()`魔法方法，
			每次调用`__next__()`方法函数会从上次暂停的地方继续执行（它会记住上次执行语句时的所有状态），
			直到再次遇到 yield 语句。这样，生成器函数可以逐步产生值，而不需要一次性计算并返回所有结果。
			当我们调用内置函数`next(生成器对象)`函数时，也会自动调用生成器对象的`__next__()`方法。

			生成器函数的优势是它们可以按需生成值，避免一次性生成大量数据并占用大量内存。
			此外，生成器还可以与其他迭代工具（如for循环）无缝配合使用，提供简洁和高效的迭代方式。
			
		
		2、生成器实现
			最简单的生成器
				提前结束生成器才会写return
				生成器就是迭代器，所以也会StopIteration异常
								
				
			把上面实现的迭代器使用生成器实现
		
		
		3、send给生成器发送数据
			|   next(生成器)         让生成器执行到下一个 `yield`，只拿值，不传值    
			|   生成器.send(None)    和 `next()` 完全等价             
			|   生成器.send(值)      给生成器传一个值，再让它执行到下一个 `yield` 并拿值
